Seletor universal

Css: *{color: red;}

Atribui a todos elementos essa característica.

Seletor de atributo

Alguns dos tipos são:

  • [attr] Se refere a uma atribuição.
  • [attr=value] Se refere a uma atribuição com o valor "value".
  • [attr*=value] Se refere a uma atribuição que tenha "value" em algum lugar.
  • [attr^=value] Se refere a uma atribuição que tenha o seu value precedido por "value".

Exemplo:

HTML
<a href="http://example.com">Texto com exemplo</a>

CSS
a[href*="example"] { color: tomato;}

Pseudo-classes e Pseudo-elementos

:first-child

Esta pseudo-classe casa com o elemento que é o primeiro filho de um outro elemento.
A regra a seguir casa com o primeiro elemento parágrafo p na div com classe article:

Primeiro paragrafo

Segundo paragrafo

CSS
div.article p:first-child {font-style:italic;}

:link e :visited

Essas pseudo-classes se referem ao acesso da página, o primeiro é um link comum e o segundo mostra se o link ja foi visitado, tendo sua cor alterada.
Apenas um pode ser usado simultaneamente.

naovisitado
visitado

CSS
a {color: green;}
a:visited {color: tomato;}

Pseudo-classes Dinâmicas

:hover

É comumente utilizado para destacar o local em que o ponteiro esta localizado sobre, mesmo sem clicar.



CSS
.botao:hover {
color: black;
background-color: white;
}

:active

Assim como o hover ele também destaca a coisa, mas agora com a necessidade de clicar.



CSS
.botao2:active {
color: black;
background-color: white;
}

:focus

Se refere a um elemento que aceita eventos do teclado.



CSS
.textbar:focus {
color: rgb(39, 134, 125);
font-weight: bold;
}

Efeito cascata e !important

Qual estilo será usado quando houver mais de um estilo especificado para um elemento HTML?

Estilo Inline

Estilo Incorporado ou o Estilo Externo, depende da ordem de declaração dentro da tag Head

Estilo padrão do navegador

Portanto, um estilo Inline (dentro de um atributo style de um elemento HTML específico) tem a prioridade mais alta, o que significa que ele substituirá um estilo definido dentro da tag <style>, ou em uma folha de estilos externa <link>, ou um valor padrão do navegador.

Ainda, além do efeito cascata, temos a especificidade dos seletores CSS.
Digamos que dois seletores CSS, definidos em uma folha de estilo externa, casem com o mesmo elemento. Qual será renderizado?
Em resumo, quanto maior o nível de detalhamento do seletor da regra CSS, maior a prioridade.
HTML
<!DOCTYPE HTML>
<html>
    <body>
      <div id="conteudo">
      <p class="nivelc">Texto do primeiro parágrafo, blá...blá..</p>
      <p>Texto do segundo parágrafo, blá...blá...</p>
      </div>
      <p>Texto do terceiro parágrafo, blá...blá...</p>
    </body>
</html>

CSS
#conteudo p.nivelc {
color: green;
}

#conteudo p {
color: blue;
font-size: 18px;
}

p {
color: red;
font-size: 26px;
}

Resultado:

Texto do primeiro parágrafo, blá...blá..

Texto do segundo parágrafo, blá...blá...

Texto do terceiro parágrafo, blá...blá...

A declaração com uso de !important faz com que aquela propriedade tenha prioridade sobre todos as demais propriedades de outros seletores iguais, independentemente da especificidade e do efeito cascata.

p {
color: red !important;
font-size: 26px;
}

Caso haja conflito, vale o efeito cascata e a especificidade.
Caso haja conflito entre a definição do desenvolvedor e do usuário, vale a do usuário.
